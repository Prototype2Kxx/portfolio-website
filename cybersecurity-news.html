About this dashboard

What Is This?

This Cybersecurity News Dashboard is a portfolio project that combines automation, AI, and front-end development to create a functional security intelligence tool. It automatically collects articles from trusted cybersecurity RSS feeds, sends each one to an AI model for processing, and displays the results in a clean, filterable dashboard — updated twice daily without any manual intervention.

System Architecture
RSS / API Sources → GitHub Actions (Python) → Claude AI → data/news.json → Website Dashboard Step-by-step: 1. Schedule Trigger — GitHub Actions runs at 07:00 and 19:00 UTC daily 2. Fetch RSS Feeds — Python reads 5 trusted cybersecurity feeds 3. Deduplicate — Skip articles already stored in data/news.json 4. AI Processing — Each new article sent to Claude Haiku API: • 2–4 sentence summary • Category classification • Severity rating (Low / Medium / High / Critical) • 3–6 tags • "Why this matters" explanation 5. Store — Updated data/news.json committed to GitHub repo 6. Deploy — File FTP-uploaded to IONOS automatically 7. Display — This page reads the JSON and renders the cards
How It Works — Step by Step
  1. GitHub Actions runs a scheduled Python script at 7 am and 7 pm UTC every day — no server required.
  2. RSS Feed Collection — the script fetches articles from five cybersecurity feeds (The Hacker News, BleepingComputer, Krebs on Security, Dark Reading, NCSC UK).
  3. Deduplication — article URLs are compared against those already in storage so nothing is processed twice.
  4. AI Summarisation — each new article is sent to Claude Haiku with a structured prompt requesting a summary, category, severity, tags, and a "why it matters" explanation.
  5. Storage & Deployment — the updated data/news.json is committed back to the GitHub repository and FTP-uploaded to the live IONOS server automatically.
