Cybersecurity News Dashboard

A fully automated security intelligence feed using RSS, Claude AI, and GitHub Actions. No server required.

View Live Dashboard ← All Projects

What Is This Project?

The Cybersecurity News Dashboard automatically collects articles from five trusted security sources, sends each one to the Claude Haiku AI model for processing, and publishes the results to a searchable, filterable card-based dashboard, updated twice every day without any manual work.

There's no backend server. Everything runs on free GitHub Actions infrastructure. The pipeline fetches, processes, and stores articles in a static JSON file, which the browser reads directly. Deployment to IONOS shared hosting happens automatically via SFTP.

How the Automation Pipeline Works

1
Scheduled Trigger via GitHub Actions

A cron job fires the workflow at 07:00 and 19:00 UTC every day. No server needed; GitHub's free CI infrastructure runs the whole pipeline automatically.

2
RSS Feed Collection

A Python script uses the feedparser library to pull the latest articles from five trusted cybersecurity sources: The Hacker News, BleepingComputer, Krebs on Security, Dark Reading, and NCSC UK.

3
Deduplication

Article URLs are checked against what's already stored in data/news.json. Anything already processed is skipped, so the AI is only called for genuinely new articles, keeping API costs minimal.

4
Claude AI Processing

Each new article's title and snippet are sent to Claude Haiku via the Anthropic API. The model returns a structured JSON response with a 2–4 sentence summary, category, severity rating (Low → Critical), up to six tags, and a "why it matters" explanation.

5
Storage & Git Commit

New articles are prepended to the existing list (capped at 50 total) and written back to data/news.json. The workflow then commits and pushes this file to the GitHub repository automatically.

6
SFTP Deployment to IONOS

The updated JSON file is uploaded via paramiko (Python SFTP library) to the live IONOS web server. The website immediately serves the new articles with no manual intervention.

Pipeline Architecture
RSS / API Sources → GitHub Actions (Python) → Claude Haiku AI → data/news.json → IONOS SFTP → Live Website Step 1 │ 07:00 + 19:00 UTC cron trigger Step 2 │ feedparser reads 5 RSS feeds Step 3 │ Deduplicate against stored URLs Step 4 │ Claude Haiku API per new article: │ • 2–4 sentence summary │ • Category classification │ • Severity (Low/Med/High/Critical) │ • 3–6 topic tags │ • "Why this matters" note Step 5 │ news.json updated → git push Step 6 │ paramiko SFTP upload to IONOS
Security & Ethics

No website scraping is used. Only public RSS feeds and APIs. Every card links to the original article. AI summaries are clearly labelled. API credentials are stored exclusively in GitHub Secrets, never in code or version history.

Front-End Dashboard Features

Challenges & Solutions

No server budget

Solved by using GitHub Actions (free for public repos) for all compute. The website is fully static; it just reads a JSON file.

IONOS uses SFTP, not FTP

The initial deployment used Python's ftplib for FTPS, but IONOS requires SFTP. Switched to paramiko with an SSH client and key-based host acceptance.

Files landing in wrong directory

IONOS's SFTP root isn't the webroot. Files need to go to /site/. Fixed by adding an IONOS_WEBROOT environment variable and prefixing all upload paths.

Keeping API costs low

Using Claude Haiku (the most cost-efficient model), capping at 10 new articles per run, and deduplicating by URL before any API call is made. Estimated cost: pennies per month.
